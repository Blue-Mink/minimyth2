Minimyth
Overview

Minimyth2 is a small Linux distribution that turns a diskless computer into a MythTV frontend.

This version of Minimyth2 is version 8.5.0.r199 and contains:

  • Minimyth2 offers following key functional enhancements:
    • Universal kernel (and infrastructure) allowing to boot Minimyth2 under BIOS and EFI systems
    • Auto-detection and auto-configuration for Nvidia/AMD VDPAU, Intel VAAPI and VMware supported graphic hardware
    • Build-in support for external user equipment control with help of LIRC irblaster
    • Build-in OSD notifications for: New Mails, Recording status, FE/BE hardware status, upcoming recordings
    • Configuration-less mode where system is able to boot without any configuration files while offering majority of it's functions.(tested on various Gigabyte/Intel/AsRock platforms with Microsoft MCE USB remote)
    • Extended sleep/resume support for fast&reliable appliance like experience
    • Enhanced MCEUSB STRAT button function supporting rescue system actions
    • Build-in MythTV frontend process watchdog
    • Enhanced networking functions like single NFS share per FE, myth /home dir as NFS share on server with auto-inicialization of remote home directory
    • Adobe Flash player auto-installation
    • MakeMKV OSS part for BluRay playback/RIP/Copy
    • Simple VoIP SIP client allowing to make/receive/voice-mail SIP calls from MythTV GUI
  • MythTV build from sources through GIT 20161115-r199(g89bc8e7) with

Detail

Minimyth2 is based on Minimyth.org project. Initially it was simple recompile of Minimyth, but with time it has evolved to support some additional functions originally not originally present in Minimyth.

Minimyth2 compared to original Minimyth has following components changed:

  • Init sub-system is based on bash. Sometime ago, Pablo decided to move Minimyth init process to perl. Major reason was speed-up of boot process. One from major assumptions for Minimyth2 is extensive usage of suspend-to-RAM sleep/resume. This results with much lower numbers of cold/warm system boots so speed of init process becomes less important. I believe any further efforts related to Minimyth2 boot area should be targeted to systemd as it allows to best reward (in terms of speed-up) for taken work (move current SystemV to systemd).
  • All game emulators (mame, snez, etc) are moved to separated software group and, for reducing image size, are excluded from Minimyth2 image. There is plan to move them to external category and finally offer them as .sfs archive.
  • PXE environment was changed from gPXE to iPXE. iPXE is actively maintained successor for gPXE and switch to iPXE is obovious move.

Minimyth2, when compared to original Minimyth, has following components added:

  • Universal kernel (and infrastructure) allowing to boot Minimyth2 under BIOS and EFI systems. Minimyth2 automatically recognizes type of HW firmware so no any configuration on Minimyth2 side is required.
  • UDisks and related required infrastructure (sgutils, atasmart, polkit, LVM, etc). As current MythTV for external media discovery relies exclusively on udisks - I decided to add it to Minimyth2.
  • VDPAU based hardware decode support for AMD/ATI graphics. Minimyth2 has latest X.org ATI driver, latest Mesa3D configured to use Gallium/LLVM driver model for AMD/ATI. This allows to have working hardware video decode on AMD/ATI cards with UVD2.2+ hardware.
  • MakeMKV OSS part needed for BluRay playback and one-press button DVD/BluRay rip/copy.
  • Simple VoIP SIP client allowing to make/receive/voice-mail SIP calls from MythTV GUI.
  • Perl packages required to supporting OSD notifications and OSD status reporting features.
  • Itegration with Netflix (via chhgle-chrome as player).

Minimyth2 compared to original Minimyth has following components removed:

  • External Media players. warpme experience with build-in MythTV is good enough that last years warpme was using it as exclusive player for all content he has. This expirience and strong desire to reduce Minimyth2 image size to possible minimum leads to decision to remove all external players (mplayer, Xine and VLC) from Minimyth2.
  • Graphics drivers and their config/support related parts for: Openchrome (VIA), Savage (SiS), Goede and FOSS Nvidia (nouveau). Currently Minimyth2 has build-in drivers only for:
    • Nvidia (closed source drivers)
    • ATI (FOSS)
    • Intel (FOSS)
    • VMware
    Build-in graphics drivers are providing support for following graphic hardware
  • Support for MythTV releases older than 0.27-fixes/Master.
